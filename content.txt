# **DevDocs — Complete Project Analysis**

## **1. CORE FEATURES**

**DevDocs** is an AI-powered README/documentation generator for GitHub repositories. It provides:

- **🔍 Stack Detection** — Automatically analyzes repo tech stack (Next.js, Django, Go, Rust, etc.)
- **📝 Intelligent Section Generation** — Creates professional README sections using AI
- **🎯 Tier-Based Access** — Anonymous (5 gen/day), Free (50 gen/day), Premium (unlimited)
- **💾 Save & History** — Users can save generated READMEs and view generation history
- **🤖 Multi-AI Provider Support** — Falls back through Groq → Gemini → OpenAI → Anthropic → Ollama
- **⚡ Rate Limiting** — DDoS protection using Upstash Redis with sliding window
- **🔐 GitHub OAuth** — Users authenticate via GitHub through Supabase
- **📊 Usage Tracking** — Per-user/session daily limits with Redis + Supabase persistence

**Use Cases:**
- Developers quickly generating professional READMEs for new projects
- Job seekers showcasing well-documented projects to employers
- Open-source maintainers creating comprehensive docs at scale

```typescript
export class AIOrchestrator {
  private providers: ProviderConfig[] = [
    { provider: 'groq', priority: 1 },      // Fastest
    { provider: 'gemini', priority: 2 },    // Good balance
    { provider: 'openai', priority: 3 },    // Most capable
    { provider: 'anthropic', priority: 4 }, // High quality
    { provider: 'ollama', priority: 5 },    // Local fallback
  ]

  async generate(prompt: string, context?: string): Promise<AIResponse> {
    // Try providers in priority order
    for (const providerConfig of this.getProvidersInOrder()) {
      try {
        const response = await providerConfig.instance.generate(prompt, context)
        return response  // Success! Return immediately
      } catch (error) {
        logger.warn(`${providerConfig.provider} failed, trying next...`)
        continue  // Circuit breaker pattern
      }
    }
    throw new Error('All AI providers failed')
  }
}
```

```typescript
export class OpenAIProvider implements AIProviderInterface {
  private client: OpenAI

  isConfigured(): boolean {
    return !!process.env.OPENAI_API_KEY
  }

  async generate(prompt: string, context?: string): Promise<AIResponse> {
    const response = await this.client.chat.completions.create({
      model: 'gpt-4o',
      messages: [
        { role: 'system', content: 'You are a technical README writer.' },
        { role: 'user', content: prompt }
      ],
      temperature: 0.7,
      max_tokens: 2048,
    })

    return {
      content: response.choices[0].message.content || '',
      provider: 'openai',
      tokensUsed: response.usage?.total_tokens,
    }
  }
}
```

### **Workflow 1: Stack Detection**

[src/lib/analyzers/index.ts](src/lib/analyzers/index.ts):

```typescript
export class StackAnalyzer {
  analyze(): DetectedStack {
    const packageJson = this.findFile('package.json')
    const requirementsTxt = this.findFile('requirements.txt')
    const goMod = this.findFile('go.mod')
    const cargoToml = this.findFile('Cargo.toml')

    let stack: DetectedStack = {
      primary: 'unknown',
      secondary: [],
      language: 'unknown',
      ...
    }

    // Detect by ecosystem
    if (packageJson) {
      stack = this.analyzeJavaScript(packageJson, stack)
    } else if (requirementsTxt) {
      stack = this.analyzePython(stack)
    } else if (goMod) {
      stack = this.analyzeGo(goMod, stack)
    } else if (cargoToml) {
      stack = this.analyzeRust(cargoToml, stack)
    }

    return stack
  }

  private analyzeJavaScript(packageJson: string, stack: DetectedStack) {
    const pkg = JSON.parse(packageJson)
    const allDeps = { ...pkg.dependencies, ...pkg.devDependencies }

    // Detect primary framework
    if (allDeps['next']) {
      stack.primary = 'nextjs'
    } else if (allDeps['react']) {
      stack.primary = 'react'
    } else if (allDeps['@nestjs/core']) {
      stack.primary = 'nestjs'
    }

    // Detect language
    stack.language = allDeps['typescript'] ? 'TypeScript' : 'JavaScript'

    return stack
  }
}
```

```typescript
export async function joinWaitlist(entry: WaitlistEntry): WaitlistResponse {
  // Atomic upsert — prevents race conditions at DB level
```

```typescript
describe('StackAnalyzer', () => {
  it('should detect Next.js with TypeScript', () => {
```

## **ARCHITECTURE STRENGTHS & EXTENSION POINTS**

### **✅ Strengths**

1. **Robust AI Provider Fallback** — If Groq fails, seamlessly tries Gemini → OpenAI
2. **Tier-Based Monetization** — Easy to upgrade anonymous → free → premium
3. **Caching Strategy** — Redis (real-time) + Supabase (persistent), prevents duplicate generations
4. **Optional Auth** — Users can generate without login; conversion path clear
5. **Stack Detection Extensible** — Adding support for new languages/frameworks is straightforward

### **🔧 Backend Overhaul Considerations**

If redesigning the backend, focus on:

1. **API Response Format Standardization** — Ensure all endpoints follow same schema
2. **Error Handling** — Centralize error codes & messages
3. **Logging/Observability** — Add structured logging for debugging generation failures
4. **Database Indexes** — Already in place; ensure query performance is monitored
5. **Rate Limiting Bypass** — Consider admin bypass or higher tiers
6. **Generation Queue** — For heavy load, consider async job queue (Bull, RabbitMQ)
7. **Provider Cost Tracking** — Log token usage per provider for analytics
8. **Webhooks** — Add webhooks for completion notifications
